PROGETO - Testes Unitários com JUnit e Gradle

COMO EXECUTAR OS TESTES
No IntelliJ IDEA:
Botão direito na classe ContaBancariaTest → Run / Executar
Ou pelo terminal: ./gradlew test


CONCEITOS APLICADOS
Neste projeto foram utilizados testes unitários com JUnit 5 (Jupiter) para verificar o funcionamento da classe ContaBancaria.

Os testes foram organizados utilizando o padrão AAA (Arrange, Act, Assert):
Arrange – preparação dos dados para o teste
Act – execução da ação que será testada
Assert – verificação do resultado esperado
O Gradle foi utilizado para fazer o gerenciamento das dependências do projeto e também para executar os testes automatizados.

Como os testes automatizados ajudam na qualidade do software?

Os testes automatizados ajudam a garantir que o sistema esteja funcionando corretamente.
Eles permitem testar o código de forma rápida e automática, verificando se as funcionalidades estão retornando os resultados esperados.

Além disso, os testes ajudam a:
identificar erros mais rapidamente
evitar que mudanças no código quebrem funcionalidades existentes
aumentar a confiança no sistema
facilitar a manutenção do software no futuro

Dessa forma, os testes contribuem para melhorar a qualidade, confiabilidade e segurança do software.
